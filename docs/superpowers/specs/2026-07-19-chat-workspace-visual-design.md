# 用户端聊天工作区视觉设计

## 1. 目标与边界

第一版用户端面向需要配置自有模型端点的中文用户，页面的唯一核心任务是：选择可用模型、发送文本、可靠观察流式生成状态，并能停止、恢复或重新生成。

布局参考 DeepSeek Chat 的低干扰工作区原则，但不复制其 Logo、品牌色、文案、图标组合或精确尺寸。公开页面当前需要登录后才能进入聊天区，因此具体结构以本项目已批准的 Conversation 契约、状态机和错误恢复流程为准。

第一版只设计浅色主题和文本对话闭环。附件、图片生成、视频生成、管理端和深色主题保留扩展位置，不在本批提供不可用控件。

## 2. 视觉原则

- **内容优先：** 助手回答使用无卡片的正文列，避免把长文本切成连续气泡。
- **状态可见：** 流式、停止、中断、失败和完成必须在消息附近表达，不只依赖全局提示。
- **控件克制：** 常驻控件只保留新建会话、模型选择、发送或停止、侧栏开关和账户菜单。
- **结构稳定：** 顶栏、消息列和输入区使用稳定尺寸，状态文字、图标和加载内容不能推动整体布局。
- **操作语义一致：** 同一动作在按钮、状态提示和结果反馈中使用相同名称。

不采用蓝紫渐变、暖米色加衬线标题、深色霓虹、装饰光斑、营销型大标题或卡片网格。这些表现与高频对话工具的工作属性无关，也会削弱用户对内容和状态的扫描效率。

## 3. 设计 Token

### 3.1 颜色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `canvas` | `#F6F8F7` | 页面画布与侧栏背景 |
| `surface` | `#FFFFFF` | 输入区、菜单和弹层 |
| `ink` | `#18211E` | 主正文与高优先级图标 |
| `muted` | `#66736E` | 次级文字、时间和占位内容 |
| `line` | `#D9E0DD` | 分隔线、输入边界和焦点外轮廓底色 |
| `pine` | `#167A68` | 主操作、流式状态和选中态 |
| `rust` | `#B54A3A` | 失败、破坏性操作和错误状态 |
| `amber` | `#8A570F` | 中断、等待恢复和警告状态；在画布上对比度为 `5.71:1` |

颜色不得依赖透明渐变制造层次。层次由边界、留白、字体权重和有限阴影表达。错误与警告除颜色外必须同时包含图标和文字。

### 3.2 字体

- 产品名：`Bahnschrift`、`Segoe UI Variable`、无衬线回退；只用于侧栏顶部的 `Autumn Wind Ai`，字号 `15px`、字重 `600`。
- 界面与正文：`Segoe UI Variable`、`Microsoft YaHei UI`、`PingFang SC`、系统无衬线；正文 `15px/1.75`，控件 `14px/1.4`。
- 技术信息：`Cascadia Code`、`JetBrains Mono`、等宽回退；只用于模型 ID、错误码和用量，不用于普通正文。

不使用随视口宽度变化的字体尺寸，字间距固定为 `0`。

### 3.3 形状与阴影

- 常规圆角：`6px`；紧凑按钮和菜单项：`4px`；不使用胶囊形文本按钮。
- 图标按钮：桌面 `36px`，移动端 `40px`，内部使用 Lucide 图标并提供 Tooltip。
- 弹层阴影：`0 12px 32px rgba(24, 33, 30, 0.12)`；页面区块不使用悬浮卡片阴影。
- 焦点环：`2px solid #167A68`，外偏移 `2px`。

## 4. 布局

### 4.1 宽屏桌面

- `1024px` 及以上显示固定左侧栏，宽 `256px`，使用 `canvas` 背景，与主区通过 `1px` 分隔线区分。
- 主区顶栏高 `56px`，左侧显示当前会话标题，中间偏左放模型选择器，右侧保留账户菜单。
- 消息阅读列最大宽 `780px`，左右内边距最小 `24px`。
- 输入区最大宽 `820px`，底部安全间距 `20px`；文本域初始高 `56px`，最大高 `200px`，增长时不改变工具按钮尺寸。
- 会话滚动只发生在消息区，侧栏和输入区保持稳定。

```text
┌────────────── 256 ──────────────┬──────────────────────────────────────────┐
│ Autumn Wind Ai      [新建]      │ 会话标题      [模型选择]          [账户] │
│─────────────────────────────────│──────────────────────────────────────────│
│ 今天                            │                                          │
│  当前会话                       │        用户消息（轻边界块）              │
│  另一会话                       │                                          │
│                                 │  ┃ 助手正文（无卡片）                    │
│ 过去 7 天                       │  ┃ 推理、状态与操作跟随回答               │
│  历史会话                       │                                          │
│                                 │                                          │
│                                 │       ┌──────────────────────────┐       │
│ [设置]                 [折叠]   │       │ 输入内容          [发送] │       │
└─────────────────────────────────┴───────┴──────────────────────────┴───────┘
```

### 4.2 中小屏

- 小于 `1024px` 时隐藏固定侧栏，顶栏左侧使用菜单图标打开全高抽屉。
- 顶栏固定高 `64px`。中央区域分为两行：上行为单行省略的会话标题，下行为唯一的模型菜单触发器；右侧保留账户菜单。
- 模型触发器最大宽度为 `min(48vw, 280px)`，长名称单行省略。触发器使用 `aria-haspopup="listbox"`，`Esc` 关闭菜单并把焦点恢复到触发器。
- 消息区左右内边距 `16px`；输入区贴近底部但保留系统安全区。
- 用户消息最大宽度 `88%`；助手消息占满阅读宽度，状态轨不侵占正文可读宽度。

```text
┌────────────────────────────┐
│ [菜单] 会话标题      [账户]│
│          [模型选择]        │
│────────────────────────────│
│           用户消息         │
│                            │
│ ┃ 助手正文                 │
│ ┃ 生成状态与操作           │
│                            │
│ ┌────────────────────────┐ │
│ │ 输入内容        [发送] │ │
│ └────────────────────────┘ │
└────────────────────────────┘
```

## 5. 标志性交互：生成状态轨

每条助手生成在正文左侧显示 `3px` 状态轨，它同时承担定位、状态和恢复入口，不作为装饰：

- `PENDING`：灰色短轨，显示“等待生成”。
- `STREAMING`：`pine` 轨道，轨道末端做一次缓慢位移动画；正文持续追加但不闪烁。
- `SUCCEEDED`：轨道收束为静态 `ink` 细线，操作区显示复制和重新生成图标。
- `STOPPED`：灰色轨道配停止图标，保留已生成正文。
- `INTERRUPTED`：`amber` 轨道配恢复说明和重新生成操作。
- `FAILED`：`rust` 轨道配错误摘要；错误码放在可展开详情，不直接占据正文。
- `replay.reset`：保持当前正文，显示短暂的“正在同步”状态；快照替换完成后恢复对应生成状态。

`prefers-reduced-motion: reduce` 时禁用轨道位移动画，只保留颜色、图标和文字状态。

助手消息容器在 `PENDING`、`STREAMING` 和 `replay.reset` 同步期间设置 `aria-busy="true"`，进入任一终态后恢复为 `false`。状态轨文字使用 `role="status"`、`aria-live="polite"` 和 `aria-atomic="true"`，只播报开始生成、停止、失败、中断、开始重连、同步完成和生成完成。`content.delta` 不逐段进入 live region，避免读屏持续打断；生成完成后，完整回答作为普通可导航正文供读屏读取。

## 6. 核心组件

### 6.1 会话侧栏

- 顶部为产品名和仅图标的新建会话按钮。
- 会话按“今天”“过去 7 天”“更早”分组，当前项使用左侧 `pine` 标记和轻背景，不使用圆角胶囊。
- 每项固定高 `40px`，标题单行省略；更多操作只在悬停、聚焦或触摸菜单中出现。
- 空列表直接显示新建会话入口，不展示功能介绍。

### 6.2 模型选择器

- 采用带图标的菜单触发器，显示模型展示名，不显示端点 URL 或凭据。
- 菜单项显示文本、视觉、文件等能力图标；能力来自 Model Registry，不根据模型名称猜测。
- 禁用或不可用模型显示明确原因，不能静默回退到其他模型。
- `1024px` 及以上触发器位于顶栏标题右侧；小于 `1024px` 时使用第 4.2 节定义的中央第二行触发器，不再提供第二个模型入口。
- 模型菜单使用可键盘导航的 Radix 选择组件；关闭后焦点回到原触发器，选择完成后触发器名称与当前模型保持一致。

### 6.3 消息列

- 用户消息使用 `canvas` 背景和 `6px` 圆角，保持与助手正文的角色差异。
- 助手消息不套卡片；Markdown 标题、列表、代码块和表格遵循阅读列宽。
- 推理内容使用可折叠区域，默认不抢占正文视觉层级；不得写入普通日志。
- 消息操作使用 Lucide 图标按钮和 Tooltip，不显示常驻长文本工具栏。

### 6.4 输入区

- 输入区是单一有边界工具面，不嵌套卡片。
- 空闲时右侧为发送图标；生成中原位置切换为停止方形图标，尺寸不变。
- `Enter` 发送，`Shift+Enter` 换行；组合输入法未结束时不得发送。
- 第一版不显示附件按钮，避免提供尚未可用的文件流程。
- 未选择模型、输入为空或存在活动提交时，发送按钮禁用并提供可访问原因。

## 7. 状态与错误

- 首次加载使用固定行高骨架，不让侧栏和输入区跳动。
- 空会话中央只显示简短问候和输入区，不使用营销式 Hero。
- 连接中断后在对应状态轨内显示重连，不使用覆盖全屏的错误页。
- `replay.reset` 必须先请求快照并替换本地内容，再继续订阅；不能把缺失事件伪装成完整重放。
- 归档、停止和重新生成保持原动作命名，Toast 只确认结果，不重复解释功能。
- 所有错误展示关联 ID；端点 URL、凭据、Authorization Header、Cookie 和内部堆栈不得进入界面。

## 8. 可访问性与响应式验收

- 所有图标按钮具有可访问名称和 Tooltip；键盘焦点始终可见。
- 正文与背景对比度满足 WCAG AA；状态不只依赖颜色。
- 宽屏至少验证 `1440×900` 和 `1024×768`，断点附近验证 `800×900` 与 `768×1024`，移动端至少验证 `390×844` 和 `360×800`。
- 文本、模型名、会话标题和错误码不得越界或覆盖相邻控件。
- Playwright 覆盖新建会话、发送、停止、失败、重连、reset、重新生成、侧栏抽屉和键盘流程。

## 9. 自我审查

本方案最初容易落入“白底加单一品牌绿”的通用 SaaS 外观。为避免这一点，最终保留三类具有业务语义的强调色，并把视觉记忆点集中到生成状态轨，而不是扩大品牌色面积。侧栏、消息和输入区使用不同结构规则，不靠卡片或阴影堆叠层级。视觉风险集中在状态轨这一处，其余区域保持安静，符合高频工作工具的使用场景。
